# Network Dynamics and Learning Homeworks

This repository contains the Python implementations used for the three Network Dynamics and Learning homeworks at Politecnico di Torino.

The code supports the submitted homework reports. The PDF reports are submitted separately and are meant to be readable independently of the code.

## Contents

The assignments cover:

- **HW1**: network flows, centrality measures, shortest paths, maximum flow, social optimum, Wardrop equilibrium, and tolls.
- **HW2**: continuous-time random walks, French-DeGroot opinion dynamics, particle/node-perspective simulations, and open queueing-style networks.
- **HW3**: SIR epidemic simulations, preferential attachment graphs, vaccination dynamics, H1N1 parameter estimation, and network games/best-response dynamics.

## Repository structure

The repository is organized by homework:

```text
network-dynamics-learning/
├── hw1/
│   ├── data/
│   ├── src/
│   ├── figures/
│   └── results/
├── hw2/
│   ├── data/
│   ├── src/
│   ├── figures/
│   └── results/
├── hw3/
│   ├── data/
│   ├── src/
│   ├── figures/
│   └── results/
├── requirements.txt
```

The `src/` folders contain executable scripts and helper modules. The `data/` folders contain the official assignment files and input data. In HW1, the traffic assignment scripts require `.mat` input files inside `hw1/data/`.

## Environment setup

Create a virtual environment from the repository root:

```bash
python -m venv .venv
```

Activate it on Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

Activate it on Linux/macOS:

```bash
source .venv/bin/activate
```

Install the dependencies:

```bash
pip install -r requirements.txt
```

## How to run the code

Run each homework from its own folder. This avoids path issues because scripts may read from local `data/` folders and write to local `figures/` and `results/` folders.

### Homework 1

Main scripts:

- `hw1/src/run_problem1.py`
- `hw1/src/run_problem2.py`
- `hw1/src/run_problem3.py`

Example:

```bash
cd hw1
python src/run_problem1.py
python src/run_problem2.py
python src/run_problem3.py
```

### Homework 2

Main scripts:

- `hw2/src/run_problem1.py`
- `hw2/src/run_problem2.py`
- `hw2/src/run_problem3.py`

Example:

```bash
cd hw2
python src/run_problem1.py
python src/run_problem2.py
python src/run_problem3.py
```

`run_problem2.py` reads some outputs generated by `run_problem1.py`, so run Problem 1 first when regenerating HW2 from scratch.

### Homework 3

Main scripts:

- `hw3/src/run_problem1.py`
- `hw3/src/run_problem2.py`
- `hw3/src/run_problem3.py`
- `hw3/src/run_problem4.py`
- `hw3/src/run_games.py`

Example:

```bash
cd hw3
python src/run_problem1.py
python src/run_problem2.py
python src/run_problem3.py
python src/run_problem4.py
python src/run_games.py
```

`run_problem4.py` performs a Monte Carlo parameter search for the H1N1 model and can take longer than the other scripts.

## Outputs

Scripts may generate numerical outputs in each homework's `results/` directory and figures in each homework's `figures/` directory.

The exact output files depend on the specific script. Do not assume that every script regenerates every final report result automatically.

## Reproducibility notes

- Random seeds are used where stochastic simulations are involved.
- Monte Carlo simulations may show small numerical variability if seeds, parameters, package versions, or execution choices are changed.
- Figures and numerical results can be regenerated by running the relevant scripts.
- HW1 traffic/network exercises require the `.mat` input files to remain in `hw1/data/`, including:
  - `flow.mat`
  - `capacities.mat`
  - `traffic.mat`
  - `traveltime.mat`
